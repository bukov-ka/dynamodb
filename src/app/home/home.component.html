About

With this site you get DynamoDB knowledge and train yourself for designing DynamoDB structure. This site contains set of tasks each of which requires you to migrate SQL DB to DynamoDB.

The main problem with studying DynamoDB is moving from relation-based paradigm to action-based paradigm. Even similar relational databases can be migrated to very different DynamoDB bases because they are used differently. We saw the same in relational DBs when a 'Full Name' field could be totally fine in one DB but the same field could violate the first normal form in another DB (if we use it to get a last name from it).

That is why each task contain not only SQL DB scheme and data itself, but list of actions to be implemented. All of the actions should be fulfilled by using only one single Global Secondary Index Query.

To form DynamoDB table you will use SQL which is a natural language designed for denormalization relational databased. Alasql is used as the SQL engine. It is pretty close to usual SQL, but it is case sensitive and supports union of sets with different attributes which is closer to Dynamodb.

Solution of each task will require:

  1. Creating SQL to transform SQL DB to DynamoDB data.
  2. Defining fields for Primary Key and Sort Key (optional).
  3. Setting query parameters for each task which will produce the desired output.

Each task has sample solution. You can see it by clicking at the Solution button(visibility). But do not press it until you tried to find a solution yourself!
